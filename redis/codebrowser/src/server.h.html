455/* The "module" object type is a special one that signals that the object
456 * is one directly managed by a Redis module. In this case the value points
457 * to a moduleValue struct, which contains the object value (which is only
458 * handled by the module itself) and the RedisModuleType struct which lists

461 *
462 * Inside the RDB file, module types are encoded as OBJ_MODULE followed
463 * by a 64 bit module type ID, which has a 54 bits module-specific signature
464 * in order to dispatch the loading to the right module, plus a 10 bits
465 * encoding version. */

480/* Each module type implementation should export a set of methods in order
